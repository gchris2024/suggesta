## Q1 - Scope decisions

When comparing my original `BRIEF.md` to what I actually built, I can see that my initial scope was mostly realistic and gave me a strong foundation. I started with a clear MVP: authentication, movie search, recommendations, and saved lists, making it easy to prioritize core features. During development, I did expand the scope slightly, especially around usability and structure. For example, I utilized page-level organization, reusable UI components, and loading states so the app felt more complete in real use. I also spent more time than expected on routing and API integration details that were not fully represented in my original plan, for example editing a saved list with the PATCH endpoint and building an API for fetching movie metadata. Even with those additions, I stayed close to the project goals and avoided feature creep by keeping the “must-have” functionality as the primary target.

## Q2 - Technical challenge
The hardest technical challenge was getting client-side routing to work reliably with React Router in a static hosting setup. In frameworks like Next.js, routing and rendering behavior feel more automatic, so moving to a Vite + React Router setup required a more explicit configuration mindset. The biggest issue was handling direct navigation and page refreshes on nested routes, where the host can return a 404 because it looks for a physical file instead of serving `index.html`. Debugging this forced me to separate application logic problems from deployment behavior, which was an important learning moment. I resolved it by aligning route definitions in the front end, validating navigation flows, and making sure the deployed environment handled fallback routing correctly. In the end, this challenge improved my understanding of how SPA routing actually works under the hood, not just how to use it in development mode.

## Q3 -- AI and vibe-coding
Generative AI was very useful in this project, but only in the right contexts. It worked best for rapid scaffolding of straightforward pages such as Landing, Login, and Register, where the patterns are common and the requirements are predictable. That speed let me focus my time on higher-value parts of the application. However, for core features like Recommendations, AI output needed heavy revision. The first drafts often mixed concerns, repeated logic, or lacked clean component boundaries, which made maintenance harder until I refactored the code myself. Looking back, I could have improved results by defining stronger architecture constraints up front, including folder boundaries, data contracts, and reusable component patterns. Overall, AI accelerated momentum, but quality still depended on deliberate engineering decisions and careful review.

## Q4 - Architecture
One clear example is when a logged-in user clicks “Save” on a movie card. On the front end, the click event triggers a function in the page/component layer that calls `savedApi.ts` with the movie payload and the user’s auth token. The API client sends a request to the backend saved route (for example, a POST to add the movie). In the backend, the request passes through `verifyToken.js`, which validates the JWT and attaches user identity to the request context. The route handler in `savedRoutes.js` then reads the movie data and user ID, applies validation, and writes the entry through the Mongoose model layer (`SavedList.js`, and related movie/user references). MongoDB stores the new saved item, and the backend returns a success response with updated list data or the new record. The front end receives that response, updates local state, and re-renders the UI so the saved movie appears immediately in the user’s list.

## Q5 - If I Had 2 More Weeks
If I had more time, I would prioritize making the movie-adding experience feel more interactive, exploratory, and rewarding. Right now, the flow is functional but linear: search, pick one item, save it, repeat. I would redesign this into a multi-option workflow where users can compare several results at once, preview richer metadata, and add multiple movies in a single action. I would also add smarter UI feedback, such as optimistic updates and inline validation so users feel immediate progress instead of waiting through a rigid sequence. On top of that, I would introduce filtering and sorting while adding items, plus lightweight recommendation prompts based on what the user has already saved. From a technical perspective, I would refactor list interactions into reusable hooks and components to keep behavior consistent across Home, Recommendations, and Saved pages. The goal would be to shift the experience from a basic pipeline into a discovery flow that feels personalized and engaging.
